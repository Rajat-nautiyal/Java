Coin Change — ind / rem memoized recursion

Each call asks "using coins up to index ind, how few coins make rem?" It has exactly two moves: not‑take (drop this coin, move to index−1) or take (keep this coin, stay on the same index, shrink rem — coins can repeat). dp[ind][rem] caches every state so it's never solved twice.
